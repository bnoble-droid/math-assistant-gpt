Math Assistant GPT

A project website for a custom mathematics assistant developed with TinyLlama, LM Studio, Docker, and AnythingLLM, alongside a reliable built-in quadratic solver for correct user-facing results.

Project Overview

This project explores the creation of a custom math assistant for quadratic equations. A fine-tuned TinyLlama model was deployed locally through LM Studio and connected to AnythingLLM using Docker. The deployment pipeline worked successfully, allowing the assistant to respond through a browser-based interface.

During evaluation, however, the TinyLlama model showed unreliable mathematical reasoning for quadratic equations. To ensure the final website remains functional and correct, this website includes a deterministic quadratic solver for the final user-facing solving feature.

What worked: Local deployment, model conversion to GGUF, LM Studio server connection, Docker setup, and AnythingLLM integration.
Important note: The custom TinyLlama assistant is part of the project implementation and evaluation, but the quadratic solver below is used to guarantee correct answers for submission and demonstration.

Accepted Input Format

Enter quadratic coefficients directly using the standard form:

ax² + bx + c = 0

For example, for:

x² - 5x + 6 = 0

use:

  • a = 1
  • b = -5
  • c = 6

This approach avoids incorrect natural-language interpretation by the small LLM model.

How to Use the System

  1. Open this website and use the quadratic solver for reliable equation solving.
  2. Enter coefficients a, b, and c from the equation ax² + bx + c = 0.
  3. Click Solve Equation.
  4. Review the generated step-by-step working and final answer.

The project also includes a custom TinyLlama-based assistant deployed locally through AnythingLLM and LM Studio. That assistant is used as part of the AI integration and evaluation, but the built-in solver is used here for correctness and reliability.

Development Process

  • Fine-tuned a TinyLlama-based math model using QLoRA on quadratic-equation data.
  • Produced training output folders including adapter, final, and merged model files.
  • Converted the merged model into GGUF format with llama.cpp.
  • Loaded the GGUF model in LM Studio and exposed it through the local API server.
  • Ran AnythingLLM inside Docker and connected it to LM Studio.
  • Tested arithmetic and quadratic equation prompts through the chat interface.

Challenges Faced

  • Docker startup and WSL-related issues during setup.
  • Python version compatibility problems during model conversion.
  • Prompt formatting and template compatibility issues between AnythingLLM and LM Studio.
  • Inaccurate mathematical reasoning from the fine-tuned TinyLlama model.

Solution Implemented

To ensure a reliable final website, a deterministic quadratic solver was added to the website itself while preserving the custom LLM deployment as part of the project’s AI workflow and evaluation.
